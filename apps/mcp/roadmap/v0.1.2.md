# mcp v0.1.2 — Visibilidad de uncats y AMAs por cliente

**Estado**: ✅ cerrada · **App**: apps/mcp · **Branch**: `mcp/00-foundation` · **Tag**: `mcp-v0.1.2`

## Norte

El operador necesita **ver los uncats y AMAs de un cliente desde el chat** (Claude Desktop).
mapi **ya expone** esa data — `GET /v1/clients/:id/transactions?category=` lista el snapshot de
`client_transactions`, donde:

- **uncats** = `category` ∈ {`uncategorized_expense`, `uncategorized_income`}
- **AMAs** = `category` = `ask_my_accountant`

El snapshot lo llena el sync de customer-support (`POST /v1/clients/:id/transactions/sync`).
El MCP no tenía ningún tool de transacciones (solo los 4 de download/listado bancario), así que el
chat no podía mostrarlos. Esta versión agrega el wrapper delgado que faltaba.

## Alcance

### Sí entra

1. **Tool `list_client_transactions`** → reenvía a `GET /v1/clients/:id/transactions`. Args:
   `clientId` (UUID, requerido), `category` (enum: uncats/AMA), `filter` (all/income/expense),
   `startDate`/`endDate` (YYYY-MM-DD). La descripción le dice al LLM cómo distinguir uncats de
   AMAs y que el snapshot se llena con el sync. Cada item ya trae date/vendor/memo/account/amount
   - la nota del cliente (`response`) si existe — todo viene del endpoint, sin lógica en el MCP.

2. **Tests Tipo A** (mock fetch): URL base sin query, `category=ask_my_accountant`, y
   `filter`+rango de fechas arman el query string correcto.

### NO entra

- Tool de sync (`POST .../transactions/sync`) — es escritura/llamada a Intuit; se agrega solo si
  el operador lo pide. Hoy el foco es **ver** lo que ya está sincronizado.
- Tools de followups / public-links / dashboards uncats — se agregan cuando se necesiten.
- Resolver clientId por nombre dentro de este tool: el endpoint exige UUID (ParseUUIDPipe). El
  chat obtiene el UUID con `list_clients` (search) y lo pasa.

## Decisiones

- **D-mcp-006** — Un solo tool `list_client_transactions` cubre uncats y AMAs reenviando el
  parámetro `category` del endpoint tal cual (no dos tools `list_uncats`/`list_amas`): es el
  wrapper más delgado (D-mcp-002), 1 tool = 1 endpoint, y el LLM elige `category` según pida
  "uncats" o "AMAs". Sin `category` trae los tres tipos.

## Verificación

- Tests Tipo A verdes (CR-mcp-010/011/012).
- **En vivo (mapi local v0.28.1, 2026-06-15):** `GET /v1/clients/<Bilia>/transactions` devolvió
  8 uncats (`uncategorized_expense`) reales con vendor/memo/monto → el tool tiene data que mostrar.
- **Pendiente operador (Claude Desktop):** reiniciar el MCP para que cargue el tool nuevo y pedir
  "muéstrame los uncats de Bilia" → el chat usa `list_client_transactions` con el clientId.
