# mcp v0.1.1 — Filtros/paginación en los tools de listado

**Estado**: ✅ cerrada · **App**: apps/mcp · **Branch**: `mcp/00-foundation` · **Tag**: `mcp-v0.1.1`

## Norte

Los tools de listado de v0.1.0 son demasiado tontos: reenvían al endpoint **sin
parámetros**, así que el LLM recibe solo la primera página y tiene que filtrar/encontrar
"a ojo". Caso real (Haiku, 2026-06-15): pidió el cliente "SRE Services" → `list_clients`
solo trae la página 1 (50 de 68), SRE está en la página 2, el modelo nunca lo recibe y hace
match difuso con el cliente equivocado. La respuesta **ya incluye `total: 68`**, pero el tool
no expone ningún parámetro para traer el resto → callejón sin salida.

mapi **ya soporta** los filtros (`GET /v1/clients?search=&page=&pageSize=` con `ilike`;
`GET /v1/clients/:id/banking/credentials?portal=` desde mapi v0.28.1). El wrapper solo no los
reenvía. Esta versión los expone.

## Alcance

### Sí entra

1. **`list_clients` gana `search`, `page`, `pageSize`** (todos opcionales) → se reenvían como
   query string a `GET /v1/clients`. `search` = encontrar por nombre (server-side `ilike`, 1
   llamada, determinista). `page`/`pageSize` = navegar / traer el resto cuando `total >`
   los devueltos. La descripción del tool le dice al LLM: "para encontrar un cliente por
   nombre usa `search`; no escanees la lista".

2. **`list_client_accounts` gana `portal`** (opcional) → se reenvía como `?portal=` a
   `GET /v1/clients/:id/banking/credentials`. Resuelve "la credencial de Chase de este
   cliente" en una llamada (filtro de mapi v0.28.1, D-mapi-BW-032).

3. **Helper `qs()`** en tools.ts: arma el query string desde los args (omite
   undefined/null/'', encodea). Sin dep.

4. **Tests Tipo A** (mock fetch): cada tool arma la URL con los query params correctos;
   sin args → URL sin query (compat con v0.1.0).

### NO entra

- Auto-paginación en el handler (loop hasta `total`). Con `search` + `pageSize` el caso
  queda cubierto; el loop se difiere si aparece un caso real que lo pida.
- Filtros nuevos en mapi (todos ya existen).
- HTTP/SSE, auth real, tools de escritura (siguen en BACKLOG / v0.2.0).

## Decisiones

- **D-mcp-005** — Los tools de listado exponen los filtros/paginación que mapi YA soporta
  (`search`/`page`/`pageSize` en clients; `portal` en credenciales). Sigue siendo wrapper
  delgado (D-mcp-002): solo reenvía query params, cero lógica. `search` es el camino primario
  para "encontrar por nombre" (determinista, server-side); `page`/`pageSize` para navegar.

## Verificación

- Tests Tipo A verdes (URLs con query correctos).
- **En vivo (operador)** en Claude Desktop: pedir "SRE Services" → el chat usa
  `list_clients` con `search` (o pagina) y lo encuentra; `list_client_accounts` con
  `portal:"chase"` devuelve solo la credencial Chase.
