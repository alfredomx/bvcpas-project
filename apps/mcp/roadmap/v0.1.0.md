# mcp v0.1.0 — Scaffold + 4 tools (stdio contra mapi local)

**Estado:** 🚧 abierta · **App:** apps/mcp · **Branch:** `mcp/00-foundation`

## Norte

Levantar el servidor MCP delgado y poder, desde un cliente MCP (MCP Inspector / Claude
Desktop) apuntando a stdio, **listar clientes/portales/cuentas** y **disparar una descarga
bancaria** sin escribir HTTP a mano. Toda la lógica sigue en mapi.

## Alcance

### Sí entra

1. **Scaffold `apps/mcp`**: `package.json` (`name: mcp`), `tsconfig.json`, dep
   `@modelcontextprotocol/sdk`, entrypoint `src/index.ts` (transporte stdio), `.env.example`.
2. **Cliente HTTP a mapi** (`src/mapi-client.ts`): `fetch` con base URL + Bearer desde env;
   helper que normaliza errores de mapi a texto legible para el tool.
3. **4 tools** registrados con JSON Schema:
   - `bank_download` → `POST /v1/banking/download`.
   - `list_clients` → `GET /v1/clients`.
   - `list_portals` → `GET /v1/banking/portals`.
   - `list_client_accounts` → `GET /v1/clients/:id/banking/credentials`.
4. **Tests (Tipo A, mockeando fetch)**: cada tool arma la request correcta (método, path,
   body, header Authorization) y mapea la respuesta/el error. Sin red real.
5. **README de uso**: cómo correrlo (`npm run mcp` / stdio) y cómo conectarlo a MCP Inspector.

### NO entra

- Transporte HTTP/SSE + deploy (→ BACKLOG / v0.2.0).
- Auth real de prod (hoy: `MAPI_JWT` pegado en env).
- Step-flow fino y tools de escritura.

## Verificación

- Tests Tipo A verdes (mock fetch).
- **En vivo (operador):** con mapi corriendo en `:4000`, conectar MCP Inspector al server
  stdio → `list_clients` devuelve los clientes; `bank_download` con
  `{client:"bilia", what:"statements", params:{from:"2026-02", to:"2026-02", save:true}}`
  → 202 con job y el PDF de febrero en `.downloads/`.
