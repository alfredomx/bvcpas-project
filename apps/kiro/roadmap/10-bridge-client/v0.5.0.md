# v0.5.0 — Comando `open_tab` (abrir pestaña + esperar load)

**Estado**: 🚧 Código + 5 tests Tipo A nuevos (suite kiro 71 verdes). Typecheck + lint limpios. **Falta**: el lado mapi que lo manda (Fase 4) + smoke EN VIVO + bump/cierre tras revisión.
**App**: kiro · **Módulo**: 10-bridge-client
**Backend asociado**: `apps/mapi/roadmap/22-bank-worker` (step-flow de descarga, Fase 4).

## Norte

Agregar el único comando que faltaba para el auto-login bancario hands-off: **abrir una pestaña
nueva en una URL**. mapi lo usa cuando NO hay una pestaña del portal abierta (caso "no existe"); el
caso "ya existe" se cubre con `list_tabs` (v0.3.0). Después mapi manda la receta de login por
`execute_dom` sobre el `tabId` que devuelve `open_tab`.

Comando **genérico** (cualquier URL, sin lógica de banco) — kiro sigue tonto, igual que `list_tabs`.

## Por qué va en el SW y no como op DOM

Una op `navigate` (`location.href`) dentro del content script **se autodestruye**: al navegar, la
página vieja (y el content script que recibió el comando) se destruye → nunca puede responder el
`result` → mapi hace 504. Por eso `open_tab` corre en el **service worker** con
`chrome.tabs.create`: el SW vive afuera de la pestaña, sobrevive a la navegación, espera el `load`
de la pestaña nueva (`chrome.tabs.onUpdated` `status:'complete'`) y recién ahí contesta. Mismo lugar
y patrón que `list_tabs` (`chrome.tabs.query`). (D-kiro-B16.)

## Alcance

### Sí entra

1. **Tipo de comando** (`types.ts`): `OpenTabCommandMessage { type:'open_tab', correlationId, payload:{url} }` en la unión `IncomingCommandMessage`; `OpenTabResult { tabId, url }` en la unión de `ResultMessage.payload`.
2. **Parse** (`bridge-client.ts` `parseIncomingCommand`): rama `open_tab` que exige `payload.url` string; inválido → `null` (se ignora sin romper el socket, D-kiro-B08).
3. **Dispatch** (`dispatcher.ts`): `openTab(url)` → `chrome.tabs.create({url, active:true})` → si la pestaña no viene ya `complete`, espera `chrome.tabs.onUpdated` `status:'complete'` (con timeout 30s y limpieza del listener) → devuelve `{tabId, url}`. Sin `tabId` de Chrome → error (capturado por el `try/catch` del dispatcher → `{error}`, no rompe el socket).
4. **Tests Tipo A** (5): parse válido/ inválido (2 en `bridge-client.test.ts`); dispatch: crea+espera onUpdated complete (ignora otra pestaña / status loading), pestaña ya complete (no espera), sin tabId → `{error}` (3 en `dispatcher.test.ts`).

### NO entra

- **El lado mapi** que manda `open_tab` (agregar al contrato del bridge `23-plugin-bridge` + orquestar navigate→login en `22-bank-worker`) → Fase 4, su propia versión de mapi.
- **`navigate`/redirect de una pestaña existente** — no se necesita: si ya hay pestaña, se usa (`list_tabs`); si no, se abre (`open_tab`).
- **Receta de login de Chase** (selectores) → vive en mapi como data (kiro tonto). Ya validada (memoria `project_chase_login_automation`): `#userId-input-field-input`, `#password-input-field-input`, `#signin-button`, URL logonbox `https://secure.chase.com/web/auth/#/logon/logon/chaseOnline`.

## Flujo (con el demo bancario)

1. mapi: `list_tabs` → ¿hay pestaña en el logonbox de Chase?
   - **Sí** → usa ese `tabId`.
   - **No** → `open_tab(url logonbox)` → kiro abre la pestaña, espera load, devuelve `tabId`.
2. mapi: `execute_dom` sobre ese `tabId` con la receta de login (waitFor → fill user → fill pass → click signin).
3. mapi: post-login, `getAllAccounts()` (live) → listar cuentas → `download_checks`.

## Decisión (D-kiro-NNN)

- **D-kiro-B16** — `open_tab` corre en el SW (`chrome.tabs.create` + espera `onUpdated` `complete`), NO como op DOM (`location.href` mataría el contexto del content script → sin `result` → 504). Genérico (cualquier URL), kiro sigue tonto. Cubre el caso "no hay pestaña"; "ya hay" lo cubre `list_tabs`.

## Verificación EN VIVO (al cerrar, con mapi Fase 4)

mapi manda `open_tab` con la URL del logonbox → kiro abre la pestaña, espera que cargue, responde
`{tabId, url}` → mapi sigue con la receta de login por `execute_dom`. Los tests Tipo A (chrome
mockeado) NO abren pestañas reales.
