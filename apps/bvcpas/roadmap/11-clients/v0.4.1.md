## Objetivo

Corregir bug post-v0.4.0 donde el sidebar mostraba 50 clientes en
vez de los 68 reales. Causa: `GET /v1/clients` pagina con
`pageSize=50` por default y v0.4.0 no mandaba el query param.

Fix mínimo: `listClients` acepta `pageSize` como argumento opcional
(default `200`, máximo permitido por mapi) y `useClients` lo pasa
explícitamente. Sin paginación real con loop — 200 cubre los casos
actuales (dev: 6 clientes, prod: 77). Cuando rebasen 200,
v0.x.x implementará loop de paginación.

---

## Alcance

### Sí entra

- **`src/modules/11-clients/api/clients.api.ts`**:
  - `listClients` acepta argumento opcional `params: { pageSize?, page?, status?, tier?, search? }`.
  - Pasa los query params al SDK (`api.GET('/v1/clients', { params: { query: ... } })`).
- **`src/modules/11-clients/api/clients.api.test.ts`**:
  - Test nuevo: cuando se llama `listClients({ pageSize: 200 })`, el
    `fetch` recibe URL con `?pageSize=200`.
  - Test nuevo: sin args, no manda query (default backend).
- **`src/modules/11-clients/hooks/use-clients.ts`**:
  - Llama `listClients({ pageSize: 200 })`.
  - QueryKey incluye el `pageSize` para invalidar correctamente:
    `['clients', { pageSize: 200 }]`.
- **`src/modules/11-clients/hooks/use-clients.test.tsx`**:
  - El mock de `listClients` ahora recibe args; los tests pasan
    porque el mock ignora args.

### NO entra

- Paginación real (loop hasta `total`). Diferida hasta que sea
  necesario.
- Filtros server-side (`status`, `tier`, `search`). El sidebar mantiene
  search local. Los filtros server-side se exponen en el wrapper
  pero no se usan desde el hook.
- Migración de `useSession` al SDK. Diferida a **v0.4.2**.

## Decisiones tomadas durante esta versión

- **D-bvcpas-029** — Sidebar usa `pageSize=200` sin paginación real.
  - Razón: 200 cubre los casos actuales (dev: 6, prod: 77). Implementar
    loop de paginación hoy es complejidad sin beneficio.
  - Consecuencia: cuando algún tenant rebase 200 clientes, el sidebar
    cortará silenciosamente. Hay que monitorear `response.total > 200`
    en alguna versión futura para alertar y agregar paginación real.
