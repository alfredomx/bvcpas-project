# v0.9.0 — Shell rediseñado: icon rail + sidebar clientes + main

## Motivación

El shell actual es topbar + sidebar de clientes + main. Cuando Alfredo
quiso meter configuraciones, settings de empresa, secciones extra
(reportes, integraciones, etc.) se dio cuenta de que no había
dónde — la sidebar lateral es solo lista de clientes, no hay un
"menú de la app" por encima.

Esta versión introduce un **left icon rail** permanente a la
izquierda del sidebar de clientes. Inspirado en QuickBooks: barra
angosta vertical con ícono + label corto por item, hover muestra
flyout a la derecha con sub-items. Los items reales se definirán en
versiones siguientes — esta versión solo monta el esqueleto con
placeholders.

## Layout final

## Hover flyout

Al pasar el mouse sobre un item, abre un `<HoverCard>` (primitive
shadcn) a la derecha mostrando un panel con:

- Título grande del item (ej. "ALL APPS").
- Lista de sub-items placeholder (ej. para "All apps": A · B · C ·
  D · E · F).

Por ahora los sub-items son strings dummy. La estructura del flyout
queda lista para conectar contenido real en próximas versiones.

`HoverCard` por defecto abre a los ~700ms — bajamos a `openDelay={150}`
para que la UX no se sienta lenta. `closeDelay={100}` para que no
parpadee al moverse entre items.

```
    └── icon-rail-items.ts         # NUEVO — config de items (placeholder)
```

`icon-rail-items.ts` exporta una lista de `IconRailItemConfig`:

```ts
export interface IconRailItemConfig {
  key: string
  label: string
  icon: LucideIcon
  group: 'top' | 'pinned' | 'bottom'
  /** Sub-items dummy del flyout. */
  flyout: { title: string; items: string[] }
}
```

Esto facilita editar/reordenar items en una sola fuente sin tocar
JSX. Cuando definamos items reales, solo cambia este archivo.
