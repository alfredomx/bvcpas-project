## Objetivo

Quitar todo el diseño visual custom de las pantallas implementadas en
v0.2.0 + v0.3.0. Conservar 100% de la funcionalidad: hooks, api,
sesión, navegación, virtualización, persistencia, fetching, validación
de forms, tests.

Resultado: pantallas con HTML estructural mínimo (flex/grid/widths
necesarios para que el layout funcione), **sin** colores, tipografías
ajustadas, paddings precisos, gradients, shadows, rounded, borders
cosméticos. Visualmente "feo" pero funcional. Base limpia para que
cuando se rediseñe, se construya identidad nueva sobre código probado.

Después de v0.3.1, todos los tests siguen verde, build pasa, lógica
intacta. Solo la presentación cambia (se vacía).

**Reglas de strip (qué se remueve):**

- Todas las clases de **color** Tailwind: `bg-*`, `text-*`, `border-*`
  (cuando son cosméticas), `from-*`, `to-*`, `via-*`, `ring-*`,
  `placeholder:*`, `hover:bg-*`, `hover:text-*`.
- **Tipografías custom**: `text-[12.5px]`, `text-[14px]`, `text-sm`,
  `font-medium`, `font-semibold`, `font-bold`, `tracking-tight`,
  `tracking-wider`, `leading-tight`, `font-mono`.
- **Espaciado cosmético**: `px-3.5`, `py-2.25`, `pl-2.25`, `gap-1.5`,
  `mb-1.5` (cuando NO son requeridos para que el layout funcione).
- **Bordes cosméticos**: `rounded-*`, `border-soft`, `border-default`,
  `border-l-[3px]` (cuando es decorativo, no estructural).
- **Sombras y efectos**: `shadow-*`, `shadow-[...]`, `transition-*`,
  `animate-*`.
- **Backgrounds**: `bg-surface-*`, `bg-brand-*`, `bg-status-*`,
  `bg-popover`, `bg-card`, `bg-gradient-*`.
- **`style={{ background: ... }}`** inline (gradient navy del topbar).
- **Iconos lucide-react decorativos**: `<Mail>`, `<Lock>`, `<Search>`,
  `<ChevronsLeft>`, `<ChevronsRight>`, `<ArrowRight>`, `<Construction>`,
  `<FileQuestion>`, `<Users>`, `<UserCircle2>`, `<LogOut>`. Se
  conservan SOLO si son la única forma de transmitir la acción (ej.
  `<LogOut>` en el menu item del avatar — sin él el "Logout" texto sigue
  ahí, así que se puede borrar).
- **Logo del topbar** (`<img>` con contenedor lavender).
- **Sparkline placeholder** del SidebarRow (los 13 segmentos
  decorativos).
- **Heat bar** del SidebarRow (el div de 3px con color).
- **VIP star** del SidebarRow.
- **Status pill + silent + uncats + monto + contacto** del SidebarRow
  (todos hardcoded placeholders sin lógica).
- **Border bottom 3px naranja** del topbar.
- **Underline 2.5px naranja** de la tab activa.
- **Toda mención** de "navy", "lavender", "brand-accent", "shadow",
  "gradient" en `globals.css`.

**Reglas de conservación (qué se queda):**

- **Layout estructural**: `flex`, `flex-col`, `grid`,
  `items-center`, `justify-end`, `gap-X` (mínimo necesario), `min-w-0`,
  `min-h-0`, `flex-1`, `shrink-0`, `overflow-hidden`,
  `overflow-y-auto`, `relative`, `absolute`, `inset-*`.
- **Dimensiones funcionales**: `h-screen`, `w-full`, `h-full`,
  `min-h-screen`, `w-72` (algún ancho razonable para sidebar — no el

- **Estructura HTML**: `<button>`, `<aside>`, `<header>`, `<nav>`,
  `<input>`, `<label>`, `<form>`, `<section>`, `<main>`,
  `role="tablist"`, `role="tab"`, `aria-selected`, `aria-current`,
  `aria-label`, `aria-hidden`, `data-testid`.
- **Lógica**: `onClick`, `onChange`, `useState`, `useEffect`,
  `useQuery`, `useVirtualizer`, `useRouter`, `useParams`,
  `usePathname`, `useSession`, hooks custom.
- **Tests**: TODOS los `*.test.ts(x)` quedan idénticos. Solo se
  ajustan donde un assertion verifica una clase/estilo específico
  (no debería haber ninguno — los tests son funcionales).

**`globals.css`:**

- Borrar todos los tokens custom (`@theme` con `--color-surface-*`,
  `--color-text-*`, `--color-brand-*`, `--color-status-*`,
  `--color-status-*-bg`).
- Borrar todos los aliases shadcn (`--color-background`,
  `--color-popover`, `--color-primary`, etc.).

## Modo de trabajo

**Strip puro.** Sin TDD-first (no hay feature nueva). Para cada
componente:

1. Lo leo y aplico las reglas de strip.
2. Corro `npm run test` para verificar que los tests siguen verde.
3. Corro `npm run typecheck` y `npm run lint`.
4. Commit del componente con mensaje
   `chore(bvcpas): strip diseño en <componente>`.

**Si un test rompe** durante el strip = el test estaba acoplado al
diseño y debo reescribirlo o agregarle nota. **Si el typecheck rompe**
= falta importar/quitar algo. **Si el build rompe** = error real, lo
arreglo antes de commit.
