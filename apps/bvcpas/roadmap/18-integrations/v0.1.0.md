# v0.1.0 — Integrations conectado a backend real

## Motivación

En v0.9.0 quedó el caparazón visual de la pantalla
`/dashboard/clients/[clientId]/integrations` con mock data
hardcoded. Esta versión la conecta a los endpoints reales del
backend para que:

- La lista de cards refleje las conexiones reales del cliente.
- Los KPIs (cuando se rehabiliten) usen `stats` del backend.
- El botón **Pause** llame al endpoint real (renombrado desde
  "Disconnect" del mock).
- El botón **Check status** valide credenciales en vivo.
- El estado **Paused** sea visible y reversible vía botón **Resume**.

El módulo `18-integrations` arranca su numeración interna en `v0.1.0`
(primer roadmap propio del módulo; la pantalla visual nació como
ampliación dentro de `roadmap/15-app-shell/v0.9.0.md`).

## Backend disponible

Verificado contra `https://dev.alfredo.mx/v1/docs-json` y reflejado
en `src/lib/api/schema.ts` (regenerado al cierre de v0.9.0).

### `GET /v1/clients/:id/integrations`

Dashboard del cliente. Status derivado en runtime desde columnas DB
(no llama APIs externas). Para validar credenciales en vivo, usar
`POST /v1/connections/:id/test`.

Response shape (`ClientIntegrationsResponseDto`):

```ts
{
  client: { id: string, legalName: string },
  stats: {
    connected: number,
    healthy: number,
    needsAttention: number,
    errors: number,
    providersInUse: number
  },
  connections: Array<{
    id: string,                                      // uuid
    provider: 'clover' | 'square',
    providerLabel: string,
    label: string | null,
    externalAccountId: string,
    authType: 'oauth' | 'api_key',
    status: 'healthy' | 'needs_reauth' | 'paused',
    statusReason: string | null,
    pausedAt: string | null,
    pausedReason: string | null,
    lastSyncAt: string | null,                       // ISO
    createdAt: string                                // ISO
  }>
}
```

### `POST /v1/connections/:id/pause`

Body: `{ reason?: string }` — opcional, texto libre.
Responses: 204 / 404 (no existe o sin write) / 409 (ya pausada).

### `POST /v1/connections/:id/resume`

Sin body. Responses: 204 / 404 / 409 (no estaba pausada).

### `POST /v1/connections/:id/test`

Valida credenciales contra el provider en vivo.
Response: `{ ok: true, message: string }`. Errores se mapean por
`code` del envelope estándar (no por `message`).

## Decisiones de diseño (confirmadas por usuario 2026-05-23)

### D1 — Botones del footer se adaptan al estado backend

Backend no expone `DELETE /v1/connections/:id`. Las acciones reales
disponibles son `pause`, `resume`, `test`. El footer de la card
cambia según el `status` actual:

| Backend status   | Botón izquierda            | Botón derecha    |
|------------------|----------------------------|------------------|
| `healthy`        | **Pause** (rojo opcional)  | Check status     |
| `needs_reauth`   | **Pause** (rojo opcional)  | Check status     |
| `paused`         | **Resume** (neutro)        | Check status     |

Notas:
- El botón "Disconnect" del mock se renombra a **"Pause"** —
  refleja la acción real del backend.
- Si en una versión futura mapi expone DELETE, se agrega como
  acción adicional (no reemplaza a Pause).

### D2 — Estado `paused` ocupa el slot del badge actual

El badge superior derecho de la card hoy muestra `Connected`. Pasa
a mostrar el label derivado del backend status:

| Backend           | Label visible        |
|-------------------|----------------------|
| `healthy`         | `Connected`          |
| `needs_reauth`    | `Re-auth needed`     |
| `paused`          | `Paused`             |

Todos en badge neutro (sin color). El `pausedReason` (si existe)
queda accesible vía tooltip sobre el badge — fuera de scope v0.1.0.

### D3 — Catálogo limitado a `clover | square`

Las 4 conexiones mock que no son clover/square (Shopify, Amazon,
PayPal, Gusto) **se eliminan** al conectar al backend real. El
mock-data.ts queda en disco como referencia visual pero ya no se
importa en componentes.

## API wrapper

`src/modules/18-integrations/api/integrations.api.ts`:

```ts
import { api } from '@/lib/api/client'
import type { components } from '@/lib/api/schema'

export type IntegrationsDashboard =
  components['schemas']['ClientIntegrationsResponseDto']
export type IntegrationConnection =
  IntegrationsDashboard['connections'][number]

export async function getClientIntegrations(
  clientId: string,
): Promise<IntegrationsDashboard> {
  const { data, error } = await api.GET('/v1/clients/{id}/integrations', {
    params: { path: { id: clientId } },
  })
  if (error) throw error
  if (!data) throw new Error('getClientIntegrations: empty response')
  return data
}

export async function pauseConnection(
  connectionId: string,
  reason?: string,
): Promise<void> {
  const { error } = await api.POST('/v1/connections/{id}/pause', {
    params: { path: { id: connectionId } },
    body: reason ? { reason } : {},
  })
  if (error) throw error
}

export async function resumeConnection(
  connectionId: string,
): Promise<void> {
  const { error } = await api.POST('/v1/connections/{id}/resume', {
    params: { path: { id: connectionId } },
  })
  if (error) throw error
}

export async function testConnection(
  connectionId: string,
): Promise<{ ok: true; message: string }> {
  const { data, error } = await api.POST('/v1/connections/{id}/test', {
    params: { path: { id: connectionId } },
  })
  if (error) throw error
  if (!data) throw new Error('testConnection: empty response')
  return data
}
```

## Hooks (TanStack Query)

### `useClientIntegrations(clientId)`

```ts
useQuery({
  queryKey: ['client-integrations', clientId],
  queryFn: () => getClientIntegrations(clientId),
})
```

Devuelve `{ data, isLoading, isError, error }`. La pantalla
implementa loading / empty / error states obligatorios
(`CLAUDE.md` regla).

### `usePauseConnection()`

```ts
useMutation({
  mutationFn: ({ id, reason }: { id: string; reason?: string }) =>
    pauseConnection(id, reason),
  onSuccess: (_, { id }) => {
    // Invalida el dashboard del cliente al que pertenece la conexión.
    // El clientId se infiere del context del componente que llama.
    queryClient.invalidateQueries({ queryKey: ['client-integrations'] })
    toast.success('Connection paused')
  },
  onError: (err) => mapError(err, 'Could not pause connection'),
})
```

`useResumeConnection` idéntico pero llama `resumeConnection`.

### `useTestConnection()`

```ts
useMutation({
  mutationFn: (id: string) => testConnection(id),
  onSuccess: (data) => {
    toast.success(data.message ?? 'Connection is healthy')
    // No invalida el dashboard — solo es un health-check puntual.
  },
  onError: (err) => mapError(err, 'Connection check failed'),
})
```

## Loading / Empty / Error states

- **Loading:** skeleton con 4 cards placeholder (mismo grid 1/4).
- **Empty:** `connections.length === 0` → card central con
  CTA "Connect your first integration" (dispara flujo OAuth — fuera
  de scope de v0.1.0, solo toast placeholder).
- **Error:** banner rojo con `error.message` + botón "Retry"
  (refetch del query).

## Migración de mock → real

1. **Step 1**: Crear `api/`, `hooks/`, `lib/status-mapping.ts`.
2. **Step 2**: Adaptar `<IntegrationsRow>` para aceptar `connection`
   con shape del backend (no del mock).
3. **Step 3**: Adaptar `<IntegrationsScreen>` para leer `useClientIntegrations`
   en vez de `INTEGRATIONS_MOCK.connections`.
4. **Step 4**: Renombrar `disconnect-dialog.tsx` → `pause-dialog.tsx`.
   Conectar confirm → `usePauseConnection` (body opcional con `reason`).
5. **Step 5**: Conectar botón "Check status" → `useTestConnection`.
6. **Step 6**: Cuando `status === 'paused'` el footer izquierdo
   muestra "Resume" (sin dialog) en vez de "Pause".
7. **Step 7**: Loading/empty/error states.
8. **Step 8**: `mock-data.ts` queda en disco solo como referencia
   visual; ya no se importa en componentes.

## Fuera de scope (queda para versiones siguientes)

- Flujo OAuth / api-key para AGREGAR conexiones nuevas — botón
  "+ New Integration" sigue mostrando toast placeholder.
- Settings sheet (Field mapping, Sync settings, Activity) — sigue
  con mock data hardcoded; backend no expone esos endpoints aún.
- KPIs (componente `<IntegrationsKpis />`) — sigue comentado con
  TODO. Se rehabilita cuando haya 5+ conexiones reales.
- Compartir conexión (`/share`) — endpoint existe (`v0.10.0` mapi)
  pero no entra a esta versión del frontend.
