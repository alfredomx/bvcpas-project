# v0.15.0 — Búsqueda global cross-cliente

**Estado**: ✅ Cerrado: 2026-06-13
**App version**: `0.14.0 → 0.15.0`
**Módulo principal**: 19-bank-accounts

## Motivación

El buscador (v0.14.0) solo filtraba **dentro** del cliente seleccionado.
Se pidió poder traer **todas las coincidencias de todos los clientes**
(ej. "todas las credenciales Chase de la firma") sin ir cliente por
cliente.

## Cómo funciona

Dos modos en la misma barra:

- **Filtro local (input):** filtra lo visible del cliente seleccionado
  (banco + cuentas + notes), como en v0.14.0.
- **Búsqueda global (botón "Buscar"):** consulta el endpoint global
  `GET /v1/banking/credentials?search=` **sin** `clientId` → coincidencias
  de todos los clientes, con `client` y `portal` embebidos. Al activarse:
  - **bloquea** el combo de clientes y el input,
  - oculta "Add bank login",
  - el botón "Buscar" se convierte en **"Limpiar"**,
  - cada card muestra **a qué cliente pertenece** (`showClient`).
- **"Limpiar":** sale del modo global, rehabilita combo e input, borra el
  término.
- Enter en el input también dispara la búsqueda global.

## Alcance del match (importante)

- **Filtro local:** banco + `account_mask`/`label`/`account_type` + notes
  (tiene las cuentas cargadas).
- **Búsqueda global:** `client.legal_name` + `portal.name` + `notes`
  (lo que indexa el backend). **NO** matchea por número/mask de cuenta —
  el backend `search` no cubre cuentas. El caso "por banco" sí funciona y
  trae las credenciales con sus cuentas.

## Decisiones

- **D1** — Modo global vía estado `globalTerm: string | null` que cambia
  los params de `useBankLogins` (`{ search }` vs `{ clientId }`) y `enabled`.
- **D2** — En modo global no se aplica el filtro local sobre los
  resultados (el backend ya filtró; re-filtrar local descartaría matches
  por nombre de cliente).
- **D3** — Combo e input se deshabilitan en modo global para dejar claro
  que estás viendo un set cross-cliente "congelado"; "Limpiar" para salir.
