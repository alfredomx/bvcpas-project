# v0.12.0 — Bank Accounts: vista por cliente + credenciales descifradas

**Estado**: ✅ Cerrado: 2026-06-13
**App version**: `0.11.0 → 0.12.0`
**Módulo principal**: 19-bank-accounts
**Backend**: bank-worker mapi v0.16.3

## Motivación

v0.11.0 armó la vista **global** (todas las credenciales de todos los
clientes de jalón). En uso real eso es demasiada información de golpe y
no es el flujo del operador. mapi v0.16.3 agregó dos cosas que habilitan
el rediseño:

1. **Endpoints por cliente**: `GET /v1/clients/:id/banking/credentials`.
2. **Credenciales descifradas en los GET**: ahora las respuestas traen
   `username`, `password`, `security_qa` en claro para copiar/pegar.

Esta versión reescribe la pantalla a **vista por cliente**: eliges un
cliente y ves solo sus credenciales, con usuario/contraseña/QA visibles
para copiar, y dentro de cada credencial sus cuentas.

## Cambios

### SDK

- `sdk:gen` regenerado contra v0.16.3. Los DTOs de credencial
  (`GlobalCredentialResponseDto`, `ClientBankAccountResponseDto`, items
  de la lista) ahora incluyen `username` / `password` / `security_qa`.

### Pantalla `/bank-accounts` (`bank-accounts-screen.tsx`)

- **Combo de cliente con buscador** (Command + Popover) como gate. La
  query **no se dispara hasta elegir un cliente** (`useBankLogins` con
  `enabled`). Reusa el endpoint global filtrado por `clientId` (ya trae
  `client`/`portal` embebidos + credenciales descifradas).
- **Buscador "Filter by bank"**: filtra las credenciales del cliente por
  nombre del portal (client-side, en vivo). Se resetea al cambiar de
  cliente. Estado "sin coincidencias" propio.
- Estados loading / empty / error dedicados; prompt "Select a client"
  antes de elegir.

### Card de credencial (`credential-card.tsx`, nuevo)

- Muestra `username`, `password`, `security_qa` vía
  `<CredentialSecretField>` (nuevo): copiar al portapapeles + toggle de
  revelar en los secretos. **El valor solo vive en memoria del
  componente** — no se persiste en localStorage ni se loguea.
- Layout: fila 1 = Username · Password · Accounts; fila 2 = Security Q&A
  (abarca 2 columnas, **sin truncar**, `break-words`) · Notes.
- **Accounts** = `<DropdownMenu>` que lista las cuentas de esa credencial
  (`····mask · tipo · label`); "Manage accounts…" abre el panel de
  gestión. Vacío → "No accounts yet".
- Acciones por card: editar / borrar credencial.

### Sheet de cuentas (`bank-login-sheet.tsx`)

- Removidos los botones **Edit login / Delete** (eran de la credencial y
  confundían). El sheet queda solo para gestión de cuentas (add / edit /
  delete account). La edición/borrado de la credencial vive en la card.

### Formulario de credencial (`bank-login-form-dialog.tsx`)

- En alta, el **cliente queda fijo** (la pantalla ya lo seleccionó):
  `lockedClientId` / `lockedClientName` ocultan el picker.
- **Notes** pasó de `Input` a `Textarea` full-width (igual que Security
  Q&A), para escribir cómodo.
- **Anti-autocompletado** (seguridad): `autoComplete="off"` en el form y
  en username; `autoComplete="new-password"` en los passwords;
  `data-1p-ignore` + `data-lpignore` en username/password. Evita que el
  navegador o un gestor inyecten el correo/contraseña del operador y se
  sobrescriba una credencial por error.

### Limpieza

- Borrados componentes huérfanos de la vista global: `bank-logins-filters.tsx`,
  `bank-logins-table.tsx`, `bank-logins-table-row.tsx`.
- Errores y delete usan el token `destructive` de shadcn (no rojos
  literales).

## Fixes durante la sesión (on-the-fly)

1. Nombre de cliente largo: truncado en el botón del combo.
2. Card demasiado espaciada → layout compacto en 3 columnas.
3. Security Q&A reubicada bajo Username y luego ampliada a 2 columnas sin

4. Notes: de texto suelto → input visible → finalmente Textarea

6. Botón "Manage" redundante removido; acceso vía "Manage accounts…" del
   dropdown.

## Decisiones de esta versión

- **D1** — La vista por cliente reusa el endpoint **global**
  (`GET /v1/banking/credentials?clientId`) en vez del per-cliente
  (`/v1/clients/:id/banking/credentials`). Razón: el global ya devuelve
  `client`/`portal` embebidos + credenciales descifradas, y evita
  duplicar wrappers/hooks. El per-cliente queda disponible para cuando se
  embeba una tab dentro del detalle del cliente.
- **D2** — Secretos descifrados solo en pantalla: copiar vía
  `navigator.clipboard`, sin localStorage ni logging (regla de seguridad
  del backend).
- **D3** — Accounts como `DropdownMenu` (lista de solo lectura) y no
  `Select`; un combo de selección no aplica para un visor.
