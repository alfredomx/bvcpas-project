## Objetivo

Al hacer click en cualquier fila de la tabla (Uncategorized o AMA's)
se abre un `<Dialog>` shadcn con el detalle de la transacción. El
operador puede:

1. Ver el detalle de la transacción (read-only).
2. Seleccionar una cuenta QBO del dropdown (cargado vía proxy Intuit).
3. Escribir una nota para el cliente.
4. Editar el texto adjunto que se concatena a la nota (`-as per
   client's notes (MM-DD-YYYY)`), guardado en localStorage por
   usuario. Es editable para que cada operador lo personalice.
5. Click "Save" — placeholder toast "Saving..." hasta que mapi
   exponga el endpoint autenticado (v0.5.6).

## Shape de QboAccount

```ts
// Del proxy POST /v1/intuit/realms/{realmId}/call con
// query "select Id, Name, AccountType from Account"
type QboAccount = {
  Id: string
  Name: string
  AccountType: string
}
```

Respuesta del proxy:
```json
{
  "QueryResponse": {
    "Account": [{ "Id": "116", "Name": "Accounts Payable (A/P)", "AccountType": "Accounts Payable" }],
    "startPosition": 1,
    "maxResults": 10
  }
}
```

## Decisiones tomadas durante esta versión

- **D-bvcpas-042** — Modal de transacción usa `<Dialog>` (no `<Sheet>`).
  - Razón: el detalle de una transacción es una acción puntual
    centrada, no un panel de configuración lateral. Dialog es más
    apropiado.

- **D-bvcpas-043** — Texto adjunto guardado en localStorage por
  usuario.
  - Key: `bvcpas.noteSuffix`.
  - Default: ` - as per client's notes`.
  - El operador edita el texto base; la fecha se concatena al
    construir el preview (`MM-DD-YYYY` del día actual).
  - Razón: cada operador puede tener su propio texto. No es
    configuración del cliente, es configuración del usuario. En el
    futuro puede subirse a un user_settings en mapi; por ahora
    localStorage es suficiente.

- **D-bvcpas-044** — Cuentas QBO vía proxy `POST /v1/intuit/realms/{realmId}/call`.
  - Sin tipos SDK (la respuesta del proxy es `unknown`). Se definen
    tipos locales `QboAccount` en `qbo-accounts.api.ts`.
  - Query: `select Id, Name, AccountType from Account` sin filtrar
    por Active porque la query requiere campo `Active` que puede no
    existir en todos los planes QBO. Se filtran en frontend las que
    tengan `Id` y `Name`.
  - Se carga al abrir el modal (queryKey `['qbo-accounts', realmId]`).
    El cache de TanStack Query evita recargar si el operador abre
    múltiples transacciones del mismo cliente.

- **D-bvcpas-045** — Botón "Save" es placeholder hasta v0.5.6.
  - Muestra toast "Coming soon — backend endpoint pending."
  - El diseño del modal queda completo para que cuando mapi exponga
    el endpoint, solo se cambie el handler del botón.

## Flujo

1. Operador está en tab Uncat. Transactions del cliente.
2. Click en cualquier fila de la tabla (Uncategorized o AMA's).
3. `<TxDetailModal>` abre con los datos de la transacción.
4. El hook `useQboAccounts(realmId)` dispara la query al proxy Intuit
   y puebla el dropdown. Si falla → dropdown muestra "Could not
   load accounts".
5. Operador escribe la nota en el textarea.
6. Preview muestra: `{nota} {sufijo} ({fecha})`.
7. Operador puede editar el sufijo en el input de abajo (se guarda en
   localStorage al cambiar).
8. Click "Save" → toast placeholder + cierra modal.
9. (v0.5.6) El botón mandará la nota al endpoint de mapi.

## Notas operativas

- `realmId` puede ser `null` si el cliente no tiene QBO conectado.
  En ese caso el dropdown muestra "QBO not connected — connect
  QuickBooks to load accounts."
- El proxy Intuit puede fallar si el token expiró. TanStack Query
  reintenta; si falla, dropdown muestra "Could not load accounts."
- El hook `useQboAccounts` usa `enabled: !!realmId` para no disparar
  si no hay realmId.
