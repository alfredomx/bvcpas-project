## Alcance

### Sí entra

- Core (`apps/mapi_v2/core`) que compila y **arranca solo, sin plugins** (`GET /v1/healthz`).
- Port pieza por pieza de la infra de mapi: config → db → queue → errores/validación/logger → qbo-client → plugin-bridge → jwt-verify.
- Plugin-loader (registro): el core descubre y monta plugins sin importarlos por nombre.
- DB propia (`mapi_v2_local`/`mapi_v2_prod`) + seed de referencia (clientes + credenciales).

### NO entra

- Cualquier plugin de talacha (el primero, `bank`, es otra unidad/roadmap).
- Identidad/multiusuario (diferido — ver [`../../README.md`](../../README.md) + BACKLOG).
- Scalar / bull-board (se montan cuando un core/plugin los pida).
