## Decisiones tomadas durante esta versión

- **D-core-001** — Sistema host+plugins: el core bootea solo y NUNCA importa un plugin por nombre. Razón: el acoplamiento de mapi venía de hardcodear cada módulo en `app.module.ts`; aquí los plugins se montan por registro. Ver `README.md` raíz.
- **D-core-002** — `mapi` congelado como demo; la infra se porta pieza por pieza, no se toca mapi.
- **D-core-003** — Stack reusado de mapi probado.
- **D-core-004** — `tsc + tsc-alias` directo (heredado mapi D-mapi-001).
- **D-core-005** — Prefijo global `/v1` (heredado mapi D-mapi-002).
- **D-core-006** — Puerto `4200` por defecto (coexiste con mapi 4000/4100).
- **D-core-007** — Nombre de paquete `mapi_v2-core` (etiqueta privada).
- **D-core-008** — El core (host) registra SOLO la conexión raíz de BullMQ (`forRootAsync`) y reexporta `BullModule`. NUNCA registra colas de dominio (`bank-download`, etc.): cada plugin declara las suyas con `BullModule.registerQueue(...)`. Coherente con D-core-001 (core ciego a plugins).
- **D-core-009** — `REDIS_URL=redis://localhost:6379/3`: índice db Redis **3** para mapi_v2, distinto del de mapi (índice 1 local). Evita que las colas de ambos proyectos colisionen en el mismo Redis local.
- **D-core-010** — `REDIS_CLIENT` (ioredis raw, port de mapi) separado de las conexiones que BullMQ abre por su cuenta. Sirve para health check / locks / cache. `RedisModule` cierra la conexión al shutdown, igual que `DbModule` con el pool de Postgres.
- **D-core-011** — `DomainError` carga su propio `status` HTTP (campo `abstract readonly status`), y `DomainErrorFilter` lo lee directo. Se descarta el mapa central `STATUS_BY_CODE` de mapi: ese mapa obligaría al core a conocer los códigos de cada plugin, violando D-core-001 (core ciego a plugins). Cada plugin declara `code` + `status` en su propia clase de error.
- **D-core-012** — `correlation_id` (port de mapi: AsyncLocalStorage + middleware) se inyecta en cada línea de log de Pino (`customProps`) y se incluye en el cuerpo JSON de cada error, para cruzar un error visto por el cliente con sus logs. El `DomainErrorFilter` se registra global en `main.ts`; el `CorrelationIdMiddleware` se aplica a `*` en `AppModule.configure`.
