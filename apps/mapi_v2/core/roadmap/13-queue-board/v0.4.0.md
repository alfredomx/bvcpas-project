# core v0.4.0 — bull-board (dashboard de colas) ✅

**Módulo:** `13-queue-board` · **Branch:** `core/13-queue-board` · **Tag:** `core-v0.4.0`

Monta bull-board en el core para ver las colas BullMQ (jobs, payload, resultado, reintentos). Port del montaje del mapi viejo (`@bull-board/api` + `@bull-board/express` v8), adaptado a cero-reach: el core no conoce las colas, las descubre por un registro que los plugins alimentan.

## Alcance

- **`QueueBoardRegistry`** (`core/src/core/queue/queue-board.registry.ts`) — `@Global` (vía `QueueModule`): `register(name)` / `list()`.
- **`QueueModule`** → `@Global`, provee + exporta el registro.
- **`main.ts`** → `setupQueueDashboard(app)`: lee el registro, resuelve cada cola (`getQueueToken`), arma `createBullBoard`, monta el router Express en `/v1/admin/queues` (**público**, sin guard). Línea `Queues` en el banner.
- **`bank-downloader`** → `BankDownloadQueueService` registra `bank-download` en su constructor (cero-reach: el plugin aporta su cola).
- Deps: `@bull-board/api`, `@bull-board/express` (^8).

### Diverge del viejo

- El viejo montaba la cola hardcodeada (`BANK_DOWNLOAD_QUEUE`) en `main.ts` y la protegía con `requireAdminJwt` (Bearer/`?token=`/cookie). mapi_v2: **registro cero-reach** (no hardcodea) + **público** (local-only, auth real diferida). D-core-028.

## Verificación (en vivo)

- `GET /v1/admin/queues` → **200** HTML, público.
- API del board lista **`bank-download`** (descubierta por el registro, no hardcodeada).
- `POST /v1/bank/download/checks` → job registrado; contadores del board: `completed: 3`.
- typecheck + lint limpios; suite **122/122** verde.
