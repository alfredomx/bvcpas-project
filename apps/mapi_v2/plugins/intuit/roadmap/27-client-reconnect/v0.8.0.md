## Alcance

### Sí entra

- `IntuitClientController` (AdminGuard): `POST /v1/intuit/clients/:clientId/reconnect` (→ authorizeUrl) y `DELETE /v1/intuit/clients/:clientId/connection` (→ borra tokens, 404 si no había).
- `IntuitOauthService.connect` guarda state JSON `{clientId, expectedRealm?}` (expectedRealm = realm actual si ya estaba conectado); `callback` valida que la compañía autorizada coincida con `expectedRealm` → si no, `INTUIT_REALM_MISMATCH` (no cambia el realm en silencio).
- Error `IntuitRealmMismatchError` (409).
- 9 unit tests nuevos (oauth-service + client-controller) — 68 total verdes.

### NO entra

## Endpoints

- `POST /v1/intuit/clients/:clientId/reconnect` → `{ authorizeUrl }`.
- `DELETE /v1/intuit/clients/:clientId/connection` → `{ deleted }` (404 `INTUIT_TOKENS_NOT_FOUND` si no había).

## Anti-mixup

`connect` recuerda el realm del cliente en el state. En el callback: si la compañía autorizada ≠ la ya ligada → `INTUIT_REALM_MISMATCH` (409). Protege a TODO connect (reconnect, dev-oauth, `/oauth/connect`). Para cambiar de compañía a propósito: desconectar primero, luego reconectar.

## Resultado del smoke en vivo (2026-06-17, GET-only, no destructivo)

Con token admin: `POST /clients/0643ae8e…/reconnect` → **201** con `authorizeUrl`. `DELETE /clients/<inexistente>/connection` sin token → **401**; con token → **404 `INTUIT_TOKENS_NOT_FOUND`** (no se borró ningún token real). El realm-mismatch se cubre por unit test (requiere completar OAuth con otra compañía).

## Decisiones tomadas durante esta versión

- **D-intuit-028** — Reconnect/disconnect **client-scoped** (`/v1/intuit/clients/:clientId/{reconnect|connection}`). Anti-mixup: el state OAuth pasa a JSON `{clientId, expectedRealm}` y el callback exige que el reconnect sea la **misma compañía** (`INTUIT_REALM_MISMATCH`, 409). Para cambiar de compañía: desconectar primero. Aplica a todo connect, no solo al reconnect.

## Notas operativas

- GET-only contra QBO: el disconnect borra la fila local de tokens (no revoca en QBO). Revocar en QBO sería una llamada extra — diferido.
- El `DELETE /v1/intuit/tokens/:clientId` (admin, v0.1.0) sigue existiendo; este nuevo `DELETE /clients/:clientId/connection` es el equivalente client-scoped.
