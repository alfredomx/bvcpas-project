# bank-credentials v0.2.0 — BankCredentialsPort ✅

**Módulo:** `11-port` · **Branch:** `bank-credentials/11-port` · **Tag:** `bank-credentials-v0.2.0`

Publica el primer **contrato plugin→plugin** real de mapi_v2 (D-core-027): `bank-downloader` necesita el usuario/contraseña descifrados de una credencial para loguearse en el banco, y el nombre del portal para elegir el adapter. En vez de importar `bank-credentials` (reach), consume un puerto publicado en el core.

## Alcance

- **`core/src/contracts/bank-credentials.port.ts`** — `BANK_CREDENTIALS_PORT` (token) + `BankCredentialsPort` (interfaz `getDecrypted(credentialId) → DecryptedBankCredential`) + el tipo de retorno. Vive en el core, neutral.
- **`BankCredentialsPortAdapter`** — implementación: descifra los secretos (EncryptionService) + adjunta `portalName` (join).
- **Module `@Global`** + `{ provide: BANK_CREDENTIALS_PORT, useExisting: BankCredentialsPortAdapter }` + `exports: [BANK_CREDENTIALS_PORT]`. Solo el token sale; servicios/repos siguen internos.
- 2 tests unit (descifra + portalName; 404).
