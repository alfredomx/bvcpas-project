# bank-credentials v0.1.0 — port del modelo de credenciales ✅

**Módulo:** `10-credentials` · **Branch:** `bank-credentials/10-credentials` · **Tag al cerrar:** `bank-credentials-v0.1.0`

Porta el modelo de **credenciales bancarias** del mapi viejo (`22-bank-worker`) a un plugin propio: catálogo de portales + logins cifrados + cuentas individuales, con CRUD y migración de datos reales. **No** porta el worker de descarga (eso es qubot).

## Alcance

### Entra

- **3 tablas** (D-bank-001): `bank_portals`, `bank_credentials`, `bank_accounts`.
- **Cifrado** de `username`/`password`/`security_qa` con `EncryptionService` del core (D-bank-002).
- **CRUD completo** (data nuestra, no API externa → sin GET-only; D-bank-007).
- **Rutas flat** `/v1/bank/*`, `clientId` como filtro/body (D-bank-003).
- **Registro** del plugin en `core/src/registry/registry.ts` (1 línea).
- **Tests** unit (TDD-first) con mocks de repos.
- **Migración** de los datos reales del prod viejo (D-bank-005), bloque final.

### No entra

- **Eventos de auditoría** → diferido (no hay `event_log` en core, D-bank-004 / BACKLOG).
- **Worker de descarga** de transacciones → qubot (BACKLOG).
- **Compartir / acceso por usuario** → diferido (BACKLOG).

## Rutas (`@Roles('admin')`)

| Recurso           | Rutas                                                                                                                                                                            |
| ----------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Portales (global) | `GET/POST /v1/bank/portals` · `GET /v1/bank/portals/:id`                                                                                                                         |
| Credenciales      | `GET /v1/bank/credentials?clientId=&portalId=&status=&search=` · `POST /v1/bank/credentials` (clientId+portalId+user/pass en body) · `GET/PATCH/DELETE /v1/bank/credentials/:id` |
| Cuentas           | `GET /v1/bank/accounts?credentialId=` · `POST /v1/bank/accounts` (credentialId+mask+type en body) · `GET/PATCH/DELETE /v1/bank/accounts/:id`                                     |

## Errores (`DomainError` con `code` + `status`)

`BankPortalNotFoundError` (404) · `BankCredentialNotFoundError` (404) · `BankAccountNotFoundError` (404) · `BankPortalNameConflictError` (409) · `BankAccountMaskConflictError` (409).
