## Alcance

### Sí entró

- **Scaffold NestJS 11** (apps/mapi/) con 4 tsconfigs (json, build, eslint, scripts), eslint flat config v9 con reglas estrictas heredadas de mapi v0.x, `.prettierrc`, `nest-cli.json`, Dockerfile multi-stage 4 etapas, `drizzle.config.ts` + `scripts/migrate.ts` standalone, `src/db/schema/index.ts` placeholder.
- **Core infra:**
  - `core/config/`: schema Zod (`NODE_ENV`, `PORT`, `LOG_LEVEL`, `DATABASE_URL`, `PUBLIC_URL`, `LOKI_URL`) + `AppConfigModule` global con validate fail-fast + `AppConfigService` tipado.
  - `core/logger/`: nestjs-pino global con pino-pretty en local, JSON en prod, silent en test. Mixin que inyecta `correlation_id` desde AsyncLocalStorage.
  - `core/db/`: `@Global()` con tokens `DB` (drizzle) y `DB_CLIENT` (postgres-js raw). Pool max 10. `DbLifecycle` con `OnApplicationShutdown` cierra el pool con timeout 5s.
  - `core/metrics/`: `MetricsService` con `Registry` prom-client + default labels `{ app: 'mapi', env: NODE_ENV }` + `collectDefaultMetrics`. `MetricsController` `GET /metrics` (sin prefijo `/v1`), excluido de OpenAPI.
- **Common:**
  - `common/correlation/`: AsyncLocalStorage + middleware que respeta header `x-correlation-id` o genera UUID v4, lo expone en response.
  - `common/decorators/public.decorator.ts`: `@Public()` para futuro AuthGuard.
  - `common/errors/`: `DomainError` abstract + `DomainErrorFilter` global con mapper `STATUS_BY_CODE` (vacío inicial) + handler para `HttpException` y unknown.
  - `common/pipes/zod-validation.pipe.ts`: pipe genérico para body/query/param.
  - `common/version.ts`: `APP_NAME` y `APP_VERSION` leídos de `package.json`.
- **HealthModule** con `GET /v1/healthz` (`@Public()`, `@ApiTags('Health')`, `@ApiOperation`, `@ApiResponse` con `HealthResponseDto` Zod). Devuelve `{ status, version, env, uptime_s, timestamp, components: { db: { status, latency_ms } } }`. Responde 503 si algún componente está down.
- **OpenAPI con Scalar:** `setupApiDocs()` en `main.ts` expone `GET /v1/docs` (UI Scalar 3-pane modern, sin Models en sidebar) y `GET /v1/docs-json`. Usa `cleanupOpenApiDoc` de nestjs-zod para convertir schemas Zod a JSON Schema.
- **`docker-compose.yaml` prod** (raíz del repo) con stack postgres + redis + mapi para Coolify. Healthchecks en los 3 servicios. Labels Traefik para que Coolify enrute el subdominio a `mapi:4100`.
- **Deploy a Coolify** en `https://mapi.kodapp.com.mx` (vía Cloudflare Tunnel + wildcard `*.kodapp.com.mx`). Auto-deploy on push a `main`. SSL terminado por Cloudflare.

## Decisiones tomadas durante esta versión

- **D-mapi-001** — `tsc + tsc-alias` directo, sin `nest build`.
  - Diverge del TDD: no (heredado explícito de mapi v0.x D-071).
  - Razón: nest+webpack falla silencioso cuando hay tsconfig paths complejos. tsc directo es más predecible y rápido.
  - Consecuencia: el script `build` encadena `clean && tsc + tsc-alias` para src/ y scripts/ por separado.

- **D-mapi-002** — Prefijo global `/v1` con `exclude: ['metrics']`.
  - Diverge del TDD: no.
  - Razón: convención Prometheus mundial: `/metrics` sin prefijo. Los demás endpoints sí van bajo `/v1` para versionado API.
  - Consecuencia: `GET /v1/healthz` y `GET /metrics` (no `/v1/metrics`).

- **D-mapi-003** — `cleanupOpenApiDoc` de nestjs-zod (no `patchNestjsSwagger`).
  - Diverge del TDD: parcial (TDD escrito antes de actualización a nestjs-zod v5).
  - Razón: `patchNestjsSwagger` fue removido en nestjs-zod v5. La función nueva `cleanupOpenApiDoc` post-procesa el documento generado por `SwaggerModule.createDocument`.
  - Consecuencia: `setupApiDocs()` llama `SwaggerModule.createDocument` → `cleanupOpenApiDoc` antes de pasar a Scalar.

- **D-mapi-004** — Scalar `layout: 'modern'` + `hideModels: true`.
  - Diverge del TDD: no (decisión UI tomada en P0.6).
  - Razón: `'modern'` activa el 3-pane (sidebar + endpoint + try-it-out inline, sin popup). `hideModels` evita que la sección "Models" llene el sidebar conforme crezcan los DTOs — los schemas siguen accesibles al expandir cada response.
  - Consecuencia: sidebar con solo tags + operations. Modelos visibles inline al expandir endpoints.

- **D-mapi-005** — Schema env vars con preprocess `emptyToUndefined`.
  - Diverge del TDD: no.
  - Razón: `.env` real tiene `PUBLIC_URL=` y `LOKI_URL=` (vacíos como placeholder). Zod `z.string().url().optional()` falla con string vacío porque "" no es URL válido. El preprocess convierte "" → undefined antes de validar, así `.optional()` cubre el caso.
  - Consecuencia: cualquier env var URL opcional usa este patrón en `config.schema.ts`.

- **D-mapi-006** — DbModule `@Global()` con tokens `DB` y `DB_CLIENT` separados.
  - Diverge del TDD: no.
  - Razón: separar el cliente raw (postgres-js, para shutdown y queries especiales) de la instancia drizzle (uso normal en repos) permite que el shutdown limpie el pool sin pasar por drizzle. `DbLifecycle` con `OnApplicationShutdown` cierra con timeout 5s.
  - Consecuencia: módulos que necesitan SQL crudo inyectan `DB_CLIENT`; los que usan ORM inyectan `DB`.

- **D-mapi-007** — Subdominio prod = `mapi.kodapp.com.mx`.
  - Diverge del TDD: parcial (TDD original asumía dominio `*.alfredo.mx`).
  - Razón: el usuario migró el dominio principal de Coolify a `kodapp.com.mx` (2026-05-03). mapi v0.x legacy sigue en `mapi.alfredo.mx` hasta que P1 (Intuit Core de bvcpas-project) migre los 77 clientes y se apague el legacy.
  - Consecuencia: env var `PUBLIC_URL=https://mapi.kodapp.com.mx` en Coolify. Cualquier doc que mencione `mapi.alfredo.mx` para bvcpas-project está desactualizada.
