# mapi v0.13.1 — Alinear shape público al admin

**Status:** ✅ Cerrada
**Branch:** `mapi/12-customer-support`
**Módulo:** `12-customer-support`
**Decisión:** D-mapi-054 — el endpoint público `GET /v1/public/uncats/:token` espeja el shape del admin `GET /v1/clients/:id/transactions` para que el frontend público pueda reusar el mismo componente de tabla.

---

## Alcance

Cambia el shape de respuesta de `GET /v1/public/uncats/:token`:

1. Cada item agrega `qbo_txn_type` (faltaba).
2. `client_note` y `responded_at` planos se mueven a `response: { client_note, completed, responded_at } | null`.
3. Wrapper agrega `total: items.length`.
4. Conserva `client { id, legal_name, transactions_filter }` (sin cambios).
5. Se omiten campos internos del admin: `realm_id`, `qbo_txn_id`, `client_id`, `qbo_account_id`, `synced_at`; y en `response`: `appended_text`, `qbo_account_id`, `synced_to_qbo_at`.

Razón: el frontend público necesita el mismo componente de tabla que el admin con datos mínimos.
