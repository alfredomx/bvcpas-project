## Alcance

### Sí entra

- **Endpoint dashboard** `GET /v1/clients/:id/integrations`:
  - Devuelve `{ client, stats, connections[] }` para la pantalla "Integrations" de un cliente.
  - **Solo lista conexiones `provider IN ('clover', 'square')`** — los providers globales (Intuit, Microsoft, Dropbox, Google) no aplican aquí (vista futura aparte).
  - Stats agregados: `connected`, `healthy`, `needsAttention`, `errors`, `providersInUse` (max 2).
  - Cliente sin Clover/Square → stats en 0 + `connections: []` (frontend renderiza empty state).
  - Auth: `@Roles('admin')` + `@UseGuards(ClientAccessGuard)`.

- **Status derivado por conexión** (3 valores — solo desde columnas DB, sin llamar APIs externas):
  - `healthy` — OAuth: `refresh_token_expires_at > now()`. API key: cred presente.
  - `needs_reauth` — OAuth: `refresh_token_expires_at < now()` o falta.
  - `paused` — operador la pausó manualmente (campo nuevo `paused_at`).

- **Validación en vivo** vía endpoint existente `POST /v1/connections/:id/test`
  (no entra en v0.14.0 — ya existe desde v0.7.0). El frontend lo llama cuando
  el operador da click al botón "Test" en una row. Mientras tanto, el
  dashboard NO consume el provider en cada GET — render instantáneo.

- **Pause/Resume manual de conexión**:
  - `POST /v1/connections/:id/pause` — body opcional `{ reason?: string }`.
  - `POST /v1/connections/:id/resume`.
  - Migration agrega columnas `paused_at`, `paused_reason` a `user_connections`.
  - Mientras esté `paused`: el dashboard la reporta así. Próximas versiones podrán
    usar el flag para que crons/workers la ignoren (no entra en v0.14.0).
  - Auth: `@Roles('admin')` + verificación de ownership (`userId` del JWT debe ser dueño,
    o tener `connection_access` write — patrón reusado de v0.10.0).

- **Tag Scalar** nuevo: `Client Management - Integrations` (sigue convención
  del grupo `Client Management` ya existente).

- **`providerLabel` mapping** (solo los relevantes hoy):
  - `clover` → `"Clover"`
  - `square` → `"Square"`
  - Futuros providers se agregan al mapping cuando entren a la vista (global o cliente).

- **Bump version** a `0.14.0`.

### NO entra (a versiones siguientes)

- **Catálogo de providers disponibles** (`availableInCatalog`) — descartado por
  el operador, no se expone.
- **Vista de providers globales** (Intuit, Microsoft, Dropbox, Google) — vista
  futura aparte (probable `/v1/integrations/global` o similar). No se diseña
  ahora; se decide cuando exista caso de uso.
- **Tab "Field mapping"** (mapeo a campos de QuickBooks) — módulo aparte futuro
  con tabla `connection_field_mappings`. v0.15.0+.
- **Tab "Sync settings"** y **"Activity"** de la imagen — diferidos.
- **`PATCH /v1/connections/:id { label }`** — editar label desde frontend.
  Hoy se setea al conectar y no es editable. Si se requiere, v0.14.1.
- **Workers/crons respetando `paused_at`** — los syncs siguen corriendo sobre
  conexiones pausadas en v0.14.0. La columna se introduce para que el dashboard
  refleje el estado; los consumers se actualizan cuando se necesite (BACKLOG).

## Schema (migración 0009)

```sql
ALTER TABLE user_connections
  ADD COLUMN paused_at timestamptz,
  ADD COLUMN paused_reason text;
```

Sin index — los queries del dashboard ya filtran por `client_id` (que sí tiene
relevancia). `paused_at` solo se consulta tras filtrar por cliente.

## Shape del response — `GET /v1/clients/:id/integrations`

```json
{
  "client": {
    "id": "uuid",
    "legalName": "Marale, Inc"
  },
  "stats": {
    "connected": 6,
    "healthy": 4,
    "needsAttention": 1,
    "errors": 1,
    "providersInUse": 5
  },
  "connections": [
    {
      "id": "uuid",
      "provider": "clover",
      "providerLabel": "Clover",
      "label": "Blanco To Go",
      "externalAccountId": "MQZZH...",
      "authType": "api_key",
      "status": "healthy",
      "statusReason": null,
      "_status_enum": "healthy | needs_reauth | paused",
      "pausedAt": null,
      "pausedReason": null,
      "lastSyncAt": null,
      "createdAt": "2026-05-01T..."
    }
  ]
}
```

**Notas:**

- `providerLabel`: nombre legible del provider (mapping fijo en código: `intuit` → `"QuickBooks"`, `clover` → `"Clover"`, etc.).
- `lastSyncAt`: hoy es siempre `null`. No tenemos campo de "última sync exitosa"
  en `user_connections`. Se difiere hasta que un módulo lo necesite y agregue
  tracking real (probable cuando entren los conectores que escriben).
- `statusReason`: texto corto explicando el status no-healthy. Ej:
  `"Token expired on 2026-05-20"`, `"Provider returned 401"`, etc.

## Decisiones a documentar (D-mapi-XXX al cierre)

- **D-mapi-052** (tentativo): Status enum del dashboard de integraciones: 3 valores
  fijos (`healthy | needs_reauth | paused`). Sin columna `status` persistida — siempre
  derivado en runtime desde columnas DB (`paused_at`, `refresh_token_expires_at`,
  `auth_type`). Lógica pura sin I/O.
- **D-mapi-053** (tentativo): Validación de credenciales **NO** ocurre en cada GET
  del dashboard. Se delega al endpoint existente `POST /v1/connections/:id/test`
  que el frontend invoca on-demand cuando el operador da clic al botón "Test"
  en una row. Mejora UX (dashboard instantáneo) y respeta rate limits del provider.
  Estado `invalid_credentials` se elimina del enum — el frontend muestra el
  resultado del test en vivo sin persistirlo.
- **D-mapi-054** (tentativo): Pause es flag informativo en v0.14.0. Los workers
  y crons no respetan `paused_at` todavía. Se actualizarán cuando el operador
  reporte un caso real de "sigue sincronizando aunque la pausé". BACKLOG.
