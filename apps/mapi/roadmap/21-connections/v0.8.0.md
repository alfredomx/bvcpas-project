# v0.8.0 — Refactor URLs + Intuit unificado a Connections + Permisos por cliente + Read/Write granular

**Estado**: 🚧
**Inicio**: 2026-05-06
**Branch**: `mapi/21-connections`
**Módulo principal**: `21-connections`
**Reemplaza**: `20-intuit-oauth/tokens/*` (los tokens migran a Connections)
**TDD ref**: [README.md](README.md)

---

## Objetivo

Refactor estructural en una sola versión, porque las piezas están entrelazadas:

1. **Reorganizar URLs** bajo principios REST nestidos (sub-recursos
   bajo su entidad padre, OAuth uniforme `<provider>/oauth/*`, vistas
   agregadas en `views/`).
2. **Unificar Intuit con Connections**: una sola tabla `user_connections`
   y un solo plugin pattern (`IProvider`) para todos los providers.
3. **Modelo de "cuenta global readonly + cuentas personales full"** con
   columna `scope_type`. Habilita futuro de cuenta `customer-service@bv-cpas.com`
   que sirve para lectura compartida.
4. **Permisos granulares por cliente** (`user_client_access`). Habilita
   futuro de operadores con acceso parcial a la cartera.

Es el último refactor estructural antes de empezar a agregar más
providers (Google, Dropbox) y más vistas (reconciliations, w-9, 1099).

## Alcance

### 1) Reorganización de URLs (Forma C)

#### Mapeo viejo → nuevo

**Auth, Users, Clients, Connections raíz, Healthcheck**: sin cambios.

**Microsoft OAuth**:

```
POST /v1/connections/microsoft/connect    →  POST /v1/microsoft/oauth/connect
GET  /v1/connections/microsoft/callback   →  GET  /v1/microsoft/oauth/callback
```

**Intuit OAuth**:

```
POST /v1/intuit/connect                  →  POST /v1/intuit/oauth/connect
GET  /v1/intuit/callback                 →  GET  /v1/intuit/oauth/callback
GET  /v1/intuit/tokens                   →  GET  /v1/intuit/oauth/tokens (admin)
POST /v1/intuit/clients/:id/reconnect    →  POST /v1/clients/:id/intuit/reconnect
DELETE /v1/intuit/tokens/:clientId       →  DELETE /v1/clients/:id/intuit/connection
POST /v1/intuit/:realmId/call            →  POST /v1/intuit/realms/:realmId/call
```

**Transactions (sub-recurso del cliente)**:

```
POST   /v1/transactions/sync             →  POST   /v1/clients/:id/transactions/sync
GET    /v1/transactions                  →  GET    /v1/clients/:id/transactions
DELETE /v1/transactions/:txId            →  DELETE /v1/clients/:id/transactions/:txId
GET    /v1/transactions/responses        →  GET    /v1/clients/:id/transactions/responses
```

**Followups (sub-recurso del cliente)**:

```
GET   /v1/followups       →  GET   /v1/clients/:id/followups
PATCH /v1/followups       →  PATCH /v1/clients/:id/followups/:period
```

**Public links (sub-recurso del cliente)**:

```
POST /v1/public/links                  →  POST /v1/clients/:id/public-links
GET  /v1/public/links                  →  GET  /v1/clients/:id/public-links
POST /v1/public/links/:id/revoke       →  POST /v1/clients/:id/public-links/:linkId/revoke
```

**Vistas (Forma C)**:

```
GET /v1/dashboards/customer-support       →  GET /v1/views/uncats        (lista global)
GET /v1/dashboards/customer-support/:id   →  GET /v1/clients/:id/uncats  (detalle cliente)
```

**Públicos por token** (sin cambios):

```
GET   /v1/public/transactions/:token              ← igual
PATCH /v1/public/transactions/:token/:txnId       ← igual
```

#### Decisiones

- **D-mapi-018**: `<provider>/oauth/*` para todos los providers.
  `connections/` retiene su valor como management cross-provider.
- **D-mapi-019**: Forma C — `views/<x>` para listas globales,
  `clients/:id/<x>` para detalle por cliente. Mismo nombre del recurso
  en ambos lados.
- **D-mapi-020**: `realms/:realmId/call` con prefijo explícito (no `:realmId/call` plano).

    scopes: text('scopes').notNull(),
    accessTokenEncrypted: text('access_token_encrypted').notNull(),
    refreshTokenEncrypted: text('refresh_token_encrypted'),
    accessTokenExpiresAt: timestamp('access_token_expires_at', { withTimezone: true }).notNull(),
    lastRefreshedAt: timestamp('last_refreshed_at', { withTimezone: true }),
    metadata: jsonb('metadata'),
    createdAt: timestamp('created_at', { withTimezone: true })
      .notNull()
      .default(sql`now()`),
    updatedAt: timestamp('updated_at', { withTimezone: true })
      .notNull()
      .default(sql`now()`),
  },
  (t) => ({

**Decisión D-mapi-021**: `clientId` nullable. Solo Intuit lo usa. Microsoft (y futuros providers por user) lo dejan NULL.

**Decisión D-mapi-022**: UNIQUE compuesto `(user_id, provider, external_account_id)`. Permite:

- Mismo realm conectado por varios users (multi-cuenta personal de Intuit).
- Mismo Outlook conectado por un solo user (UNIQUE evita duplicados).

### 4) `ClientAccessGuard`

NestJS guard que se aplica a **todo controller que reciba `:id` del cliente** en path:

```ts
// Pseudocódigo del guard
canActivate(context) {
  const userId = jwt.userId
  const clientId = req.params.id
  const has = await db.exists(
    'SELECT 1 FROM user_client_access WHERE user_id = ? AND client_id = ?',
    [userId, clientId]
  )
  if (!has) throw new ClientNotFoundError(clientId)  // 404 (no leak)
  return true
}
```

**Decisión D-mapi-024**: Devuelve **404** si no hay acceso (no 403). No leak de existencia. El error es `CLIENT_NOT_FOUND` que ya existe.

### 5) Lógica read/write en `ConnectionsService`

```ts
// Read: prefiere personal del user actual; fallback a global readonly
async findActiveForRead(provider, clientId, userId) {
  // 1. Buscar conexión personal del user
  const personal = await query(WHERE user_id = userId AND provider AND client_id AND scope_type='full')
  if (personal) return personal
  // 2. Fallback a global readonly (la primera disponible)
  return query(WHERE provider AND client_id AND scope_type='readonly' LIMIT 1)
}

// Write: SOLO personal del user actual
async findActiveForWrite(provider, clientId, userId) {
  const personal = await query(WHERE user_id = userId AND provider AND client_id AND scope_type='full')
  if (!personal) throw new IntuitPersonalConnectionRequiredError(clientId)
  return personal
}
```

**Decisión D-mapi-025**: Error nuevo `INTUIT_PERSONAL_CONNECTION_REQUIRED` (HTTP 403) cuando un endpoint write no encuentra conexión personal.
