# v0.23.0 — Read verbs (count/list) sobre las primitivas

**Estado**: ✅ Implementado + verificado en vivo (Bilia/Chase 2026-06-14: checks/list=8, deposits/list=1, statements/list=17, sin bajar imágenes). **392 unit verdes**, typecheck + lint limpios.
**Módulo**: 22-bank-worker · **Branch**: `mapi/22-bank-worker`
**Depende de**: v0.22.0 (primitivas `searchTransactions` / `listStatements`).

## Norte

Verbos de **lectura** (preview) para el conector: contar/listar cheques, depósitos o statements
**sin descargar imágenes ni PDFs**. Baratos. Habilitan el flujo "¿cuántos cheques tiene bilia esta
semana?" antes de bajar (lo barato primero, lo caro solo si confirmas). Reusan las primitivas de
v0.22.0 — cero lógica nueva del banco.

## Alcance

### Sí entra

1. **`POST /v1/clients/:id/banking/download/checks/list`** `{ credentialId, accountMasks[], range|from+to }`
   → `searchTransactions(CHECK)` por cuenta → `{ accounts: [{ account_mask, count, items }], total }`.
   `items` = actividad cruda (`sequenceNumber, date, amount?, checkNumber?`), **sin imágenes**.

2. **`POST .../deposits/list`** — igual con `searchTransactions(DEPOSIT)`.

3. **`POST .../statements/list`** `{ credentialId, accountMasks[], yearsBack? }`
   → `listStatements({ yearsBack })` por cuenta → `{ accounts: [{ account_mask, count, items }], total }`.
   `items` = `{ documentId, date }`, **sin PDFs**. `yearsBack` default 1.

4. **Service**: `listChecks` / `listDeposits` (vía `listActivity` privado) + `listStatementRefs`. Reusan `resolveAdapter` / `resolveRange`. **Sin event_log** (son lecturas, no mutaciones).

5. **DTOs**: `ListActivitySchema`/`ListActivityResponseDto` (compartido checks+deposits), `ListStatementRefsSchema`/`ListStatementRefsResponseDto`. Permiso `banking.read`.

6. **Tests Tipo A** (3): `listChecks` (busca CHECK, items+count+total, **no** llama `downloadImage`), `listDeposits` (type DEPOSIT), `listStatementRefs` (respeta `yearsBack`, **no** llama `downloadStatementPdf`).

### NO entra

- `count` como verbo separado — el `total`/`count` ya viene en `list`. El conector deriva "cuántos" del `total`.
- Eventos / persistencia de las lecturas.

## Decisiones

- **D-mapi-BW-026** — Un solo verbo de lectura por tipo (`*/list`), que trae **count + items**; "cuántos" se deriva del `total`. No se hace un `count_*` aparte (duplicaría llamadas).
- Las lecturas **no descargan imágenes/PDF** (solo `searchTransactions`/`listStatements`) → baratas; el conector previsualiza antes de bajar.
