# v0.16.2 — Response DTOs tipados en OpenAPI (bank-worker)

**Estado**: ✅ Cerrada.
**Branch**: `mapi/22-bank-worker-v0.16.2`.

## Problema

El frontend (bvcpas) genera su SDK automáticamente desde `/v1/docs-json`. Los
endpoints de bank-worker declaraban el endpoint pero **no la forma de la
respuesta**, así que el SDK devolvía `data: never` (sin tipos) para llamadas
como `api.GET('/v1/banking/credentials')`.

**Causa raíz**: las respuestas en `dto/bank-worker.dto.ts` eran `interface` de
TypeScript (se borran al compilar — OpenAPI no las ve), y los `@ApiResponse`
de los controllers no llevaban `type:`.

## Fix

1. **`dto/bank-worker.dto.ts`** — las 5 respuestas pasan de `interface` a schema
   Zod + clase `createZodDto` (patrón estándar del repo, igual que las request
   DTOs y que `ClientDto` en 11-clients). Se conserva el `type` exportado de cada
   una (`z.infer<...>`) para que los services sigan compilando sin cambios:
   - `BankPortalResponseSchema` → `BankPortalResponseDto` + `BankPortalListResponseDto`
   - `ClientBankAccountResponseSchema` → `ClientBankAccountResponseDto` + `ClientBankAccountListResponseDto`
   - `GlobalCredentialResponseSchema` → `GlobalCredentialResponseDto`
   - `ListGlobalCredentialsResponseSchema` → `ListGlobalCredentialsResponseDto`
   - `BankAccountResponseSchema` → `BankAccountResponseDto` + `BankAccountListResponseDto`

   Los wrappers `...ListResponseDto` modelan el envoltorio `{ data: X[] }` que
   devuelven los endpoints de lista.

2. **Controllers** — `type:` agregado al `@ApiResponse` de cada respuesta de
   éxito (200/201) en los 4 controllers:
   - `bank-portals.controller.ts`
   - `client-bank-accounts.controller.ts`
   - `bank-accounts.controller.ts`
   - `bank-credentials-global.controller.ts`

## Resultado

`/v1/docs-json` ahora exporta la forma de cada respuesta. Tras regenerar el SDK,
el frontend obtiene tipos reales en lugar de `data: never`.

## Verificación

- `npm run typecheck` limpio (los services siguen compilando contra el `type`
  inferido de cada schema).

- Lógica de servicios/repositorios: ninguna.
- Rutas / contratos de request: ninguno.
