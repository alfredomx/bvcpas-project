## Alcance

### Sí entra

Cierre completo del módulo `10-core-auth` según TDD. Esta versión cubre TODOS los CR-auth y SMK-auth definidos en el TDD. No hay v0.2.x posteriores planeadas — el módulo cierra entero con esta versión.

Resumen del entregable:

- Tablas `users` y `user_sessions` con migration + trigger updated_at + seed admin inicial.
- Tabla `event_log` (compartida con futuros módulos, pero arranca aquí).
- Servicios core: `PasswordService` (bcrypt), `JwtService` wrapper, `SessionsService` (Redis + DB), `EventLogService`.
- Guards globales: `JwtAuthGuard`, `RolesGuard`. Decorators `@Roles()`, `@CurrentUser()`. Reuso `@Public()` de `00-foundation`.
- 12 endpoints auth + admin documentados en Scalar.
- 10 errores de dominio mapeados.
- 12 tipos de eventos `event_log`.
- Tests Tipo A (~30 CR) + Tipo B (~7 smoke).
- Rate limit en `/v1/auth/login` con `@nestjs/throttler`.

### NO entra (fuera de alcance)

- Recovery por email, 2FA, OAuth social, multi-tenant, rol bookkeeper, UI de login, audit avanzado de IP geolocation. Todo esto está en "NO entra" del TDD del módulo.
- v0.2.x patches: ninguno planeado. Si algo crítico aparece post-cierre, abre v0.2.1.

  - Constructor inyecta `DB`, `REDIS_CLIENT`, `JwtService`.
  - `create(user, userAgent?, ip?): Promise<{ token: string, session: Session }>`:
    - Genera UUID `jti`.
    - INSERT en `user_sessions` con expires_at calculado.
    - Firma JWT con `{ sub: user.id, email, role, jti, exp }`.
    - Retorna ambos.
  - `verify(jti: string): Promise<{ userId, role, status }>`:
    - Cache Redis primero (TTL 30s, key `session:<jti>`).
    - Si miss, query DB con JOIN `users`.
    - Lanza `SessionNotFoundError` si no existe.
    - Lanza `SessionRevokedError` si `revoked_at IS NOT NULL`.
    - Lanza `SessionExpiredError` si `expires_at < now()`.
    - Lanza `UserDisabledError` si `users.status='disabled'`.
    - Si OK, escribe cache Redis.
  - `revoke(jti: string): Promise<void>`:
    - UPDATE `user_sessions SET revoked_at = now()` para ese jti.
    - DEL cache Redis.
    - Dispara evento `auth.session.*` (decisión: el caller dispara, no el service genérico).
  - `revokeAllForUser(userId: string): Promise<number>`:
    - UPDATE batch + DEL cache para cada jti.
    - Retorna count.
  - `touchLastSeen(jti: string): Promise<void>`:
    - Debounce 5min: solo actualiza si última actualización >5min.
    - Implementación: query last_seen_at + UPDATE solo si difiere.

## Decisiones tomadas durante esta versión

- **D-mapi-009** — Scripts de línea de comando (`migrate.ts`, `seed-admin.ts`) no se compilan al `dist/`.
  - Diverge del TDD: parcial (TDD no lo prohibía pero el scaffold inicial sí los compilaba).
  - Razón: el seed necesita importar el schema Drizzle de `src/db/schema/users.ts` para tener tipos. Con `tsconfig.scripts.json` configurado con `rootDir: ./scripts`, ese import cross-rootDir rompe el `tsc -p tsconfig.scripts.json` del build. Como los scripts siempre se corren con `tsx` desde `package.json` (`db:migrate`, `db:seed`) y nunca desde Docker (las migrations se aplican manual fuera del container, D-015 mapi v0.x heredado), compilarlos a `dist/scripts/` no aporta valor.
  - Consecuencia:
    - Script `build` ahora solo compila `src/` (no `scripts/`).
    - Script `tsconfig.scripts.json` se elimina (ya no se usa para compilar).
    - Validación de tipos de scripts sigue cubierta por `npm run typecheck` (tsc --noEmit con tsconfig.json raíz que sí incluye scripts vía include).
    - `Dockerfile` no se modifica (ya copiaba solo `dist/` que ahora será solo `src/` compilado).
