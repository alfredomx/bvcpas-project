**Estado**: ✅ Implementado — **389 unit verdes**, typecheck + lint limpios.

## Norte

El service `ClientsService.resolve()` y `confirmAlias()` + la tabla `client_aliases` ya existían
desde v0.21.0, **pero no tenían endpoint HTTP** — el `clientId` se pasaba a mano (UUID). Esta
versión los **expone** para que el conector (LLM/MCP) pueda decir "bilia"/"moy"/"summit" y
desambiguar. Es la pieza 2 del stack (ver `22-bank-worker/v0.22.0.md`): barata, el código ya estaba.

## Alcance

### Sí entra

1. **`POST /v1/clients/resolve`** `{ q }` → `ResolveClientResponse`:
   - `{ status: 'resolved', via: 'alias'|'match', client }` — 1 cliente (alias exacto o 1 match difuso).
   - `{ status: 'ambiguous', candidates: [...] }` — varios (el conector pregunta cuál; ej. "summit" → Summit Group / Summit Geomatics).
   - `{ status: 'not_found' }` — ninguno.
   - Permiso `clients.read`.

2. **`POST /v1/clients/aliases`** `{ alias, clientId }` → `{ alias, client }`:
   - Upsert del alias (normalizado a minúsculas) → cliente. La próxima vez `resolve` pega directo.
   - El conector lo llama **solo** cuando la resolución fue única (no para términos ambiguos como "summit" — D-mapi-BW-020 / decisión del operador 2026-06-15).
   - Permiso `clients.update`. 404 si el cliente no existe.

3. **DTOs** (`clients.dto.ts`): `ResolveClientSchema`/`ResolveClientResponseDto`, `ConfirmAliasSchema`/`ConfirmAliasResponseDto`. `ClientSchema` se exporta para reusarlo en las respuestas.

4. **Tests Tipo A** (`clients.service.spec.ts`): `resolve` (alias / 1 match / 0 / 2+ ambiguo) + `confirmAlias` (upsert normalizado / ClientNotFoundError). +6 tests (→ 389 total).

### NO entra (diferido)

- **Auto-guardar alias**: lo decide el **conector**, no el endpoint. Regla: `confirmAlias` solo se llama cuando `resolve` dio `resolved` único (término no ambiguo). El endpoint es neutral (guarda lo que le pidan).
- **Filtrar por acceso del usuario** en `resolve`: hoy resuelve sobre todos los clientes (operador admin único). Si entra multi-usuario con acceso parcial, se filtra — diferido.

## Decisiones

- **D-mapi-clients-001** — `resolve` y `aliases` son endpoints de colección (`POST /v1/clients/resolve`, `POST /v1/clients/aliases`), no bajo `:id`. La desambiguación (status `ambiguous` + candidatos) la maneja el conector preguntando; el backend solo resuelve.
- Auto-alias = política del conector (solo en `resolved` único), no del endpoint.
